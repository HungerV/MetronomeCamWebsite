## 🚀 部署步骤

### 1. 更新配置
```bash
# 替换所有 yourusername 为实际 GitHub 用户名
# 替换所有 contact@example.com 为实际邮箱
# 替换所有 yourdomain.com 为实际域名（如有）
```

### 2. 推送到 GitHub
```bash
git init
git add .
git commit -m "Initial commit: MetronomeCam website"
git remote add origin https://github.com/yourusername/metronomecam-website.git
git branch -M main
git push -u origin main
```

### 3. 启用 GitHub Pages
1. 进入仓库 Settings > Pages
2. Source: main branch, / (root)
3. 点击 Save
4. 等待部署完成（1-2 分钟）

### 4. 验证部署
- 访问网站 URL
- 检查所有资源加载正常
- 测试所有链接和功能
- 运行 Lighthouse 测试

## 💡 技术决策

### 为什么选择纯静态技术？
- ✅ 最快的加载速度
- ✅ 最简单的部署流程
- ✅ 零运行成本
- ✅ 最高的安全性
- ✅ 最好的 SEO 表现

### 为什么不使用框架？
- ✅ 避免不必要的复杂性
- ✅ 减少文件大小
- ✅ 提高加载性能
- ✅ 降低维护成本
- ✅ 更好的浏览器兼容性

### 为什么使用 CSS 变量？
- ✅ 减少代码重复
- ✅ 易于主题定制
- ✅ 提高可维护性
- ✅ 现代浏览器原生支持
